# پیک (Payk) — پیام‌رسان امنِ بدون سرور

> نام «پیک» موقتی است (واژه‌ی کهن فارسی برای پیام‌بر/قاصد). آزادانه تغییرش بدهید.

یک پیام‌رسان متن‌محور که سه ویژگی بنیادی دارد:

1. **بدون سرور مرکزی** — هر گوشی خودش گره (node) است؛ پیام مستقیم بین دو دستگاه رد و بدل می‌شود.
2. **بدون ثبت‌نام** — هیچ شماره تلفن یا ایمیلی لازم نیست. هویت شما یک **جفت‌کلید رمزنگاری** است که مثل کیف پول رمزارز، از یک عبارت بازیابی (seed) ساخته می‌شود.
3. **بازبینی‌ناپذیر** — تنها جایی که متن خوانا (plaintext) وجود دارد، دو سرِ گفتگوست. هر چیزی در میانه — و حتی روی خودِ گوشی — فقط داده‌ی رمزشده است.

نقشه‌ی کامل را در [`DESIGN.md`](./DESIGN.md) بخوانید. هسته‌ی رمزنگاری (Dart خالص، آزموده) در [`packages/payk_core`](./packages/payk_core) آغاز شده است.

## ایده در یک نگاه

| پیام‌رسان‌های معمول | پیک |
|---|---|
| سرور مرکزی نگه‌دارنده‌ی پیام و حساب | هر گوشی، گره‌ی خودش است |
| ثبت‌نام با شماره/ایمیل | هویت = جفت‌کلید از روی عبارت بازیابی |
| اعتماد به اپراتورِ سرویس | اعتماد فقط به ریاضیاتِ رمزنگاری |
| فراداده‌ی متمرکز و قابل احضار | هیچ حساب یا سرورِ قابل‌احضاری وجود ندارد |

## خانواده‌ی فکری

پیک تنها نیست؛ روی شانه‌ی پروژه‌های واقعی و آزموده‌شده ایستاده است:
**Briar**، **Tox**، **Jami** (همتا‌به‌همتا)، **Session** و **Nostr** (هویتِ کلید-محور)،
و استانداردهای **BIP39** (عبارت بازیابی)، **Signal** (X3DH + Double Ratchet) و
**Noise Protocol Framework** (دست‌دهیِ امن). جزئیات در `DESIGN.md`.
