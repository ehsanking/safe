# tincan_core

موتورِ رمزنگاری و پروتکلِ پیام‌رسانِ **Tincan** — یک پکیجِ **Dart خالص** (بدون وابستگی به
Flutter) که قابلِ تست و ممیزی به‌صورتِ مستقل است. اپِ Flutter (Android + Windows) به این
پکیج وابسته می‌شود، اما خودِ هسته به UI وابسته نیست.

برای معماری و مدلِ تهدید، `DESIGN.md` در ریشه‌ی مخزن را ببینید.

## این فاز چه چیزی را پیاده کرده است (فاز ۱: هسته‌ی رمزنگاری)

| ماژول | کار | وضعیت |
|---|---|---|
| `Bip39` | تولید/اعتبارسنجی عبارتِ بازیابی BIP-0039 و مشتقِ seed (وردلیستِ انگلیسیِ vendor-شده و checksum-شده) | ✅ آزموده با بردارهای رسمیِ Trezor |
| `Identity` | مشتقِ قطعیِ کلیدهای Ed25519 (امضا) و X25519 (توافق) از seed با HKDF؛ امضا/تأیید؛ ECDH | ✅ |
| `ShortCode` | اثرانگشتِ هویت، کدِ ۱۰ رقمیِ کاربرپسند (با رقمِ کنترلیِ Luhn)، و «عددِ امنیتیِ» ۶۰ رقمی برای تأییدِ دستی | ✅ |
| `AeadCipher` / `XChaCha20Poly1305Aead` | رمزنگاریِ احرازشده (XChaCha20-Poly1305) پشتِ یک interfaceِ قابلِ‌تعویض | ✅ |
| `Transport` / `InMemoryNetwork` | seamِ ترابری + پیاده‌سازیِ درون‌حافظه‌ای برای تستِ سرتاسری بدونِ شبکه | ✅ |
| `SignalAccount` | حسابِ Signal با هویتِ **قطعی از همان seed** + تولیدِ pre-key و bundle (X3DH) | ✅ |
| `SecureSession` | نشستِ E2EE با **X3DH + Double Ratchet** (forward secrecy و post-compromise) | ✅ |
| `OutboundQueue` | صفِ تحویلِ «نگه‌داری روی فرستنده + تلاشِ مجددِ ۵–۱۰ دقیقه‌ای تا ACK» با jitter | ✅ |

**هنوز پیاده نشده (فازهای بعد):** ترابریِ واقعیِ P2P (`dart_libp2p`: mDNS/DHT/عبور از NAT)،
ذخیره‌سازیِ رمزشده (`drift` + `sqlite3mc`)، و پوسته‌ی Flutter (Android + Windows).

## پشته‌ی کتابخانه‌ها (بر پایه‌ی تحقیق)

- **`cryptography_plus`** (Dart خالص، نگه‌داری‌شده) — Ed25519, X25519, XChaCha20-Poly1305,
  HKDF, PBKDF2, Argon2id, BLAKE2.
- **`crypto`** — SHA-256/512 همگام برای اثرانگشت و checksumِ BIP39.
- BIP-0039 را خودمان پیاده کرده‌ایم (نه وابستگیِ ثالثِ کهنه) تا تنها ورودیِ مورد‌اعتماد،
  وردلیستِ vendor-شده‌ی تأییدشده باشد.

## اجرا و تست

```bash
cd packages/tincan_core
dart pub get
dart test        # ۴۰ تست
dart analyze     # بدون خطا
```

> در محیطِ توسعه‌ی ابری که Dart از یک پراکسیِ TLS عبور می‌کند، پیش از `pub get` این را
> تنظیم کنید: `export SSL_CERT_FILE=/root/.ccr/ca-bundle.crt`.

## تصمیم‌های امنیتیِ کلیدی

- **کدِ ۱۰ رقمی فقط راحتی است، نه لنگرگاهِ امنیت.** ۱۰ رقم ≈ ۳۳ بیت؛ امنیت از تأییدِ
  کلیدِ کامل (QR یا عددِ امنیتی) می‌آید. (مثلثِ زوکو — `DESIGN.md` §4.2)
- **عبارتِ بازیابی ۱۲ کلمه است، نه ۱۰.** ساختارِ checksumِ BIP39 فقط ۱۲/۱۵/۱۸/۲۱/۲۴ کلمه
  را می‌پذیرد؛ «۱۰ کلمه» نمی‌تواند BIP39ِ معتبر باشد. تصمیمِ ۱۰-در-برابر-۱۲ به‌عنوان یک
  انتخابِ محصول باز است (`DESIGN.md` §4.1).
- **AEAD پشتِ interface است** تا مسیرِ سخت‌سازی (routing به libsodiumِ آزموده از طریق FFI)
  بعداً بدونِ تغییرِ فراخوان‌ها ممکن باشد.

## محدودیت‌های شناخته‌شده

- **بدونِ ممیزیِ امنیتیِ مستقل.** کتابخانه‌های Dart این primitiveها هنوز به‌طورِ رسمی ممیزی
  نشده‌اند؛ پیش از تولید، AEAD/Argon2id را به libsodium مسیردهی کنید.
- **نرمال‌سازیِ NFKD** برای عبارتِ بازیابی (ASCII، بی‌مسئله) انجام می‌شود، اما passphraseِ
  غیر-ASCII فعلاً همان‌طور که هست استفاده می‌شود.
- این فاز هنوز **forward secrecy**ِ کاملِ هر-پیام (Double Ratchet) ندارد؛ تستِ دودِ فعلی از
  یک کلیدِ نشستِ مشتق‌شده از ECDH استفاده می‌کند که جانشینِ موقتِ آن مرحله است.
