# tincan_net

ترابریِ **همتا‌به‌همتا** برای پیام‌رسانِ Tincan، ساخته‌شده روی `dart_libp2p`. این پکیج
interfaceِ `Transport` از `tincan_core` را پیاده می‌کند، پس هسته هیچ‌گاه مستقیم با libp2p
حرف نمی‌زند و این لایه قابلِ‌تعویض می‌ماند.

## چرا یک پکیجِ جدا؟

دو دلیل:

1. **ایزولاسیونِ ریسک.** `dart_libp2p` تک‌نویسنده، کم‌مخاطب و بدونِ ممیزی است. با نگه‌داشتنش
   پشتِ interfaceِ `Transport`، اگر روزی لازم شد می‌توان آن را با WebRTC یا ترابریِ دیگری
   عوض کرد بدونِ دست‌زدن به هسته.
2. **تضادِ وابستگی.** `dart_libp2p` به `protobuf ^3.x` نیاز دارد، اما جدیدترین
   `libsignal_protocol_dart` (0.8.x) به `protobuf ^6`. تنها نسخه‌ای از libsignal که با
   libp2p جمع می‌شود **0.7.2** (روی protobuf 3) است؛ به همین دلیل `tincan_core` دقیقاً به
   آن pin شده. (پروتکلِ Signal در این نسخه‌ها تغییری نکرده.)

## وضعیت

| قابلیت | وضعیت |
|---|---|
| ترابریِ **TCP** + امنیتِ **Noise** + muxۀ **Yamux** | ✅ آزموده روی loopback واقعی |
| `Libp2pTransport` (پیاده‌سازیِ `Transport`): send/inbound روی stream | ✅ |
| peer-id **قطعی از seed** (آدرسِ پایدار) | ✅ |
| یکپارچگی با `OutboundQueue` | ✅ آزموده سرتاسری |
| ترابریِ **UDX** + سوراخ‌کردنِ NAT (hole-punching) | ⏳ گامِ بعد |
| کشفِ **mDNS** (شبکه‌ی محلی) | ⏳ گامِ بعد |
| مسیریابیِ **DHT** (دفترچه‌تلفنِ غیرمتمرکز) | ⏳ گامِ بعد |

## مدلِ فریم

هر فریم روی یک streamِ کوتاه‌عمر و روی یک پروتکلِ واحد (`/tincan/msg/1.0.0`) می‌رود:
باز کردن → نوشتنِ `[طولِ ۴ بایتی][بایت‌ها]` → بستن. آدرسِ همتا به‌صورت
`"<peerIdBase58>|<multiaddr,...>"` کدگذاری می‌شود — کافی برای dial، و برای فریم‌های ورودی
هم ساخته می‌شود تا گیرنده بتواند پاسخ بفرستد.

## اجرا و تست

```bash
cd packages/tincan_net
dart pub get
dart test     # دو host واقعی روی TCP، تبادلِ فریم رمزشده‌ی Noise
```

> در محیطِ ابری: پیش از `pub get` مقدارِ `SSL_CERT_FILE=/root/.ccr/ca-bundle.crt` را تنظیم
> کنید.

## نکته‌ی صداقت

- آدرس‌های همتا هنگامِ `send` **مستقیم** به peerstore اضافه می‌شوند، چون `connect` آن‌ها
  را از یک addrs-factory عبور می‌دهد که آدرس‌های loopback/خصوصی را فیلتر می‌کند.
- TCP اول انتخاب شد چون جهان‌شمول و قابل‌اتکاست؛ برای عبور از NATِ اپراتورها به UDX/
  hole-punching نیاز است که گامِ بعدی است.
